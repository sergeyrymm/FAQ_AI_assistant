## FAQ-ассистент с RAG (FAISS) и OpenAI через ProxyAPI

Пример проекта FAQ-ассистента для сайта: данные FAQ → RAG через FAISS → backend на FastAPI → виджет чата на фронтенде.

Ассистент отвечает на вопросы клиентов о почтовом сервисе Mail.Ru.

При использовании ИИ-Ассистента нагрузка на отдел техподдержки снижается на 50%.

Прямая оплата для юрлиц (с р/с) и ИП - только за то, что реально спросили.

### Структура проекта

- `backend/`
  - `app.py` — FastAPI-сервер, эндпоинт `/chat`, интеграция с OpenAI через ProxyAPI и поиск по FAISS.
  - `build_index.py` — скрипт построения RAG-индекса из FAQ-данных.
  - `rag_index.py` — функции загрузки индекса и поиска похожих ответов.
- `data/`
  - `faqs.json` — (опционально) пример данных FAQ (вопрос–ответ).
  - `KnowledgeDB.md` — база знаний в markdown формате.
  - `faiss_index.bin`, `faqs_metadata.npy` — будут созданы скриптом `build_index.py`.
- `frontend/`
  - `index.html` — пример простой главной страницы.
  - `assist.html` — страница с описанием проекта и виджетом чата.
- `requirements.txt` — зависимости Python.

### Подготовка окружения

1. Установите Python 3.10+.
2. В корне проекта создайте файл `.env`:

```env
OPENAI_API_KEY=ваш_ключ_openai
```

3. Установите зависимости:

```bash
pip install -r requirements.txt
```

### 1. Подготовка данных и построение RAG-индекса (FAISS)

Отредактируйте `data/faqs.json` под свой бизнес (список объектов `{ "question": "...", "answer": "..." }`).

Затем запустите скрипт построения индекса:

```bash
python -m backend.build_index
```

Скрипт:
- посчитает эмбеддинги для текстов `вопрос + ответ` через ProxyAPI(OpenAI) (`text-embedding-3-small`);
- создаст FAISS-индекс `faiss_index.bin`;
- сохранит метаданные вопросов/ответов в `faqs_metadata.npy`.

### 2. Запуск backend-сервера

```bash
uvicorn backend.app:app --reload --host 0.0.0.0 --port 8000
```

Эндпоинты:
- `POST /chat` — основной чат с RAG:
  - вход: JSON `{ "message": "текст вопроса", "top_k": 3 }`;
  - выход: `{ "answer": "...", "context": [ { "question": "...", "answer": "..." }, ... ] }`.
- `GET /health` — проверка статуса.


### Как это работает (коротко)

1. **Данные FAQ** (`data/faqs.json`) → скрипт `backend/build_index.py` создаёт эмбеддинги и FAISS-индекс.
2. **Запрос пользователя** → backend (`/chat`) получает текст, считает эмбеддинг, ищет ближайшие FAQ в FAISS.
3. **RAG**: найденные FAQ передаются в `system + user` промпт для `gpt-4.1-mini` в OpenAI через ProxyAPI.
4. Модель генерирует короткий ответ на русском, используя контекст FAQ, и возвращает его во фронтенд-виджет.
